Why this matters

Moving the threshold left catches anomalies faster but risks flagging normal signals as false alarms. Moving it right reduces false alarms but risks missing real jamming/interference. The chart on the left shows exactly where the real error distributions sit for each class, so the tradeoff can be reasoned about with data instead of guessing. The currently deployed threshold (0.00183) was chosen at 4 standard deviations above the clean-signal mean, which happens to fully separate all three classes in this evaluation set -- but that separation is a property of this dataset, not a guarantee, which is exactly why this page exists.
